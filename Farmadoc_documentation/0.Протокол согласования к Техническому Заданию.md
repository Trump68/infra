## 1. Цель документа

Зафиксировать перечень изменений к действующему ТЗ (Приложение № 1 к извещению №024/2026), которые:
- не меняют предмет закупки и целевую функциональность;
- уточняют архитектурные и приемочные формулировки;
- синхронизируют ТЗ с текущими решениями технического проекта (`TZ/1. Пояснительная записка к техническому проекту.md`).

## 2. Ключевые основания для корректировок

По действующему ТЗ (PDF) закреплены:
- API Gateway на базе Kong;
- сервис аутентификации на базе Keycloak;
- векторная база данных как Redis/Pinecone (или аналог).

По `TZ/1. Пояснительная записка к техническому проекту.md` дополнительно уточнены:
- API Gateway: Kong **или функциональный аналог**;
- IdP: Authentik (базовый вариант) или Keycloak по согласованию;
- целевой ландшафт хранения/обмена: Milvus, PostgreSQL, S3 MinIO, RabbitMQ, Redis при необходимости.

## 4. Предлагаемые редакционные замены (фрагменты)

### 4.1. Для пункта 4.1.1.3 (Сервис аутентификации)

Заменить формулировку:
- было: интеграция на базе Keycloak;
- предлагается: интеграция через IdP (Authentik как базовый вариант реализации, Keycloak как допустимая альтернатива по согласованию с Заказчиком).

### 4.2. Для пунктов 4.1.1.4–4.1.1.5 (RAG и векторная БД)

Уточнить используемые хранилища и роли компонентов без изменения состава функций:
- Milvus - векторный индекс;
- PostgreSQL - операционные данные и метаданные;
- S3 MinIO - документы и артефакты;
- RabbitMQ - асинхронный обмен.

### 4.3. Для пункта 4.1.1.2 (API Gateway)

Заменить формулировку:
- было: API Gateway (Kong);
- предлагается: API Gateway на базе Kong или функционального аналога.

## 5. Оценка влияния на предмет закупки

Предлагаемые корректировки:
- **не изменяют** предмет закупки и функциональный класс ПО;
- **не расширяют** состав закупаемых функций;
- уточняют только формулировки по API Gateway, IdP и контуру хранения/обмена.
