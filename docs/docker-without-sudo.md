# Запуск Docker без sudo

Чтобы запускать Docker без `sudo`, добавьте своего пользователя в группу `docker`.

## Шаги

1. **Добавьте пользователя в группу `docker`:**
   ```bash
   sudo usermod -aG docker $USER
   ```

2. **Примените новую группу** (один из вариантов):
   - **Вариант А:** выйти из системы и войти снова (рекомендуется).
   - **Вариант Б:** в текущей сессии выполнить:
     ```bash
     newgrp docker
     ```
     После этого в этой вкладке терминала Docker будет работать без sudo.

3. **Проверка:**
   ```bash
   docker run hello-world
   ```
   Команда должна выполниться без `sudo` и без ошибок.

## Примечание

Группа `docker` даёт права, близкие к root (полный доступ к хосту через контейнеры). Добавляйте в неё только доверенных пользователей.

## Ошибка `KeyError: 'ContainerConfig'` при `docker-compose up`

Возникает при использовании **Docker Compose V1** (`docker-compose` с дефисом) с новыми образами.

**Вариант 1 — перейти на Compose V2 (рекомендуется):**
```bash
docker compose up -d
```
(пробел между `docker` и `compose`). Если команды нет, установите: `sudo apt install docker-compose-v2`.

**Вариант 2 — удалить старый контейнер vllm и поднять заново:**
```bash
docker rm -f vllm 2>/dev/null
docker-compose up -d
```
